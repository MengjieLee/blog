数据开发

知识储备

这会是一个逐步深入，不断积累知识点和经验的过程

编程语言

  • Go: 高性能计算和分布式系统开发
  • Python: 数据处理和分析

线性数据结构

  • Array: 连续存储，随机寻址高效（O(1)），插入删除低效（O(n)）
    • 分布式计算框架 Hadoop MapReduce、Spark 中用到 Array 存储批量数据，如日志切片、数值型特征等
    • 结合向量化运算，如 Spark 的 Vector 操作优化数据处理效率
  • Linked List: 非连续存储，随机寻址低效（O(n)），插入删除高效（O(1)）
    • 内存管理中模拟链表结构，如 Spark 的 内存缓存链，动态管理数据块
    • 分布式系统的元数据管理，如 HDFS 的 Block 分片，记录文件分块的位置信息

树形数据结构

  • Binary Tree: 每个结点最多两个子结点，常见类型包括 Binary Search Tree、AVL Tree、Red-Black Tree 等
    • 分布式索引构建，如 Elasticsearch 的倒排索引 Inverted Index 基于红黑树优化
    • 数据排序与查找，在 Spark 的 sortByKey 中，底层可能用树结构优化排序性能
    • 数据库成本优化器(CBO) 使用决策树进行词法分析，生成最优执行计划
  • Heap: 完全二叉树，分为最大堆（父>=子）和最小堆（父<=子）
    • 海量数据 Top-N 问题，如用最小堆筛选日志中的高频词（MapReduce 流程 Map → Combine(Optional) → Shuffle → Reduce 中的 Combine 阶段的局部 Top-N），在 Map 端提前局部聚合并减少网络 IO 开销
    • 资源调度，YARN 的调度器基于最大堆从堆顶取出优先级最高的任务去分配资源
    • Spark Streaming 的事件时间窗口（Event Time Window）用堆管理超时数据

集合与映射结构

  • Set: 元素唯一、无序（哈希集合）或有序（树集合）
    • 去重处理，如 Spark 的 distinct() operation 算子对海量用户 ID 去重（底层基于哈希集合）
    • 交集/并集计算，在用户分群场景中，用集合运算快速计算不同群体的重叠部分
    • Flink 的状态管理中，用集合存储已处理的事件 ID，避免重复处理
  • Map: 键值对存储，哈希表实现（O(1) 查找）或树结构实现（有序映射）
    • HBase、Redis 的核心数据结构
    • MapReduce 的 Shuffle 阶段通过键分组数据（如按用户 ID 分组统计）
    • Kafka 的 Consumer Group 管理中，用映射记录每个分区的消费偏移量（Offset）

队列与栈

  • Queue: 先入先出（FIFO），常见实现有循环队列、优先队列
    • 消息中间件（如 Kafka）的消息存储与消费队列
    • 任务调度，如 Spark 的 DAG 调度器用队列管理待执行的 Task
    • Flink 的流式处理中，事件按时间顺序进入队列，保证处理顺序性
  • Stack: 后入先出（LIFO），常见实现有数组、链表
    • 字符串提取成对的小括号、大括号、方括号
    • 解析 SQL 语句的语法树
    • 分布式事务中的撤销操作，如 HBase 的 Write-Ahead Logging（WAL）回滚机制
    • Spark 的 RDD 依赖链（Lineage）通过栈结构记录转换历史，支持容错恢复

扩展数据结构（大数据场景特化）

  • Bloom Filter: 布隆过滤器，概率型数据结构，用位数组和哈希函数判断元素是否存在（可能误判，不会漏判）
    • 海量数据去重优化， 如 HBase 用布隆过滤器减少磁盘 IO（判断 RowKey 是否存在于某 Region）
    • 垃圾邮件过滤、爬虫 URL 去重等场景
    • Flume 的拦截器（Interceptor）中用布隆过滤器过滤已处理的日志条目
  • BitMap: 位图，用二进制表示元素存在性，节省存储空间
    • 用户行为统计，记录用户是否点击某商品
    • 倒排索引优化，搜索引擎中用位图快速计算多个关键词的交集文档
    • Spark 的 BitSet 类型用户高效存储布尔型特性（如用户标签）

数据结构预算法的延伸能力

  • 算法思维：
    • 排序算法：快速排序、归并排序、Spark 的 sortBy 基于 Timsort
    • 查找算法：二分查找在 HDFS 块定位中的应用
    • 分布式算法：如一致性哈希，解决数据分片与节点动态扩展问题，如 Kafka 分区分配、Redis 集群均会用到
  • 性能优化：理解数据结构的时间/空间复杂度，例如：
    • 用哈希表代替列表实现高频查询
    • 用跳表（Skip List）代替平衡树（如红黑树）实现有序数据结构（Redis 的有序集合 ZSET 基于跳表）
  • 框架底层原理：
    • Spark 的 RDD 依赖关系基于有向无环图（DAG），需理解图的遍历算法(BFS/DFS)
    • Hadoop 的 MapReduce 洗牌阶段涉及分区(Partition)、排序(Sort)、合并（Merge），对应数据结构的组合使用
    • 重点理解数据结构在分布式场景下的改造，如分布式哈希表 DHT、分片存储策略

这些知识点是数据开发的基石，能帮助你更高效且合理地设计数据管道、优化分布式作业

快速开始

Spark 2.0 版本之前, Spark 的核心编程接口是弹性分布式数据集(RDD)。Spark 2.0 版本之后, RDD 被 Dataset 所取代, Dataset 跟 RDD 一样也是强类型的, 但是底层做了更多的优化。Spark 目前仍然支持 RDD 接口, 你可以在 RDD 编程指南 页面获得更完整的参考，但是我们强烈建议你转而使用比 RDD 有着更好性能的 Dataset。想了解关于 Dataset 的更多信息请参考 Spark SQL, DataFrame 和 Dataset 编程指南。
